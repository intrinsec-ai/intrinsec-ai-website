Each session used to reset. Classification-based identity and certs give an agent a name — not a memory. Without memory, there's no real identity to protect or compare against. The Identity Plane gives each agent one persistent memory, so the full trajectory is always visible: who drifted, what changed, and whether it matters.

SIEM-less Detection

Fleet-Wide Threats.
No Data Lake.

A fleet of agents and users. Each request looks normal alone. Together, they're a coordinated campaign — and every agent forgets the others. Intrinsec gives the fleet one shared memory: inbound intent is compressed across all agents so coordinated threat patterns surface without a SIEM, a data lake, or any hard storage.

  • ◆ One shared memory for the fleet — compresses millions of raw prompts into a single modelable signal across every agent
  • ◆ Novel threat patterns — coordinated campaigns surface even when each individual request passes every check
  • ◆ No SIEM, no data lake, no hard storage — memory lives with the agent, not in external infrastructure

See the Identity Plane in Action

A lightweight reference implementation as a plugin for OpenClaw — the popular self-hosted AI agent gateway. Observational by design.

@intrinsec-ai/openclaw-identity-plane

Identity Watchdog for OpenClaw Agents

Calibrates against your agent's cognitive files (SOUL.md, AGENTS.md, etc.) to build an identity anchor — a model of what your agent is supposed to do. Every inbound message is scored against that anchor. Drift is visible on a live dashboard — nothing is blocked, nothing is intercepted.

What it does
  • Identity calibration from cognitive files
  • Per-message drift scoring
  • Live drift dashboard with time-series chart
  • Identity compaction — LLM audit narratives grounded in named anchors
  • File-change annotations on the drift timeline
What it isn't
  • Lossless — subtle injection in long context can average out
  • Multi-tenant or fleet-scale
  • Persistent across long horizons
  • Multimodal
